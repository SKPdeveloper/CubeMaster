# Дизайн: Premium Glassmorphism 2.0 (нова дизайн-система CubeMaster)

Дата: 2026-07-09

## Контекст і мотивація

Поточна дизайн-система (`ui/theme/`, `ui/components/`) має накопичені проблеми:
- "Glassmorphism" реалізований лише через напівпрозорі градієнти (`GlassCard.kt`) — без справжнього розмиття вмісту позаду картки.
- Мертві константи `glassSurfaceLight/Dark` у `Color.kt` — оголошені, але ніде не використовуються.
- У незакомічених WIP-правках (відкинуто, засташовано в `git stash` під назвою `old-glassmorphism-design-wip`) колір `gold` помилково прирівняли до `red` (`#E30A17`), хоча в закомічених ресурсах застосунку (іконка) це два різні бренд-кольори — `#CE1126` (червоний) і `#C9A227` (золотий).
- `TabularNumberStyle` не застосовується повністю в `SummaryScreen.kt` — береться лише `fontFamily`, ігнорується `fontFeatureSettings = "tnum"`, тому суми кошторису не вирівнюються по цифрах.
- Екрани (`ProjectsScreen`, `SummaryScreen`, `LayersScreen` та інші) змішують дві різні візуальні мови картки — `Surface(color = surfaceVariant)` і `GlassCard` — на одному екрані.
- `OrnamentalDivider` (зигзаг-ромби) виглядає декоративно, не "технологічно".

minSdk піднімається з 29 до 31 (Android 12) — рішення users, ухвалене свідомо, щоб мати доступ до `RenderEffect`/апаратного backdrop blur без software-фолбеку для старіших пристроїв.

## Обсяг

1. Дизайн-система: `Color.kt`, `Theme.kt`, `GlassCard.kt`, `OrnamentalDivider.kt`, `CommonComponents.kt`, `Type.kt`.
2. Ключові екрани: `ProjectsScreen`, `RoomsScreen`, `GeometryScreen`, `SummaryScreen`/кошторис.
3. Нова залежність: `dev.chrisbanes.haze` (Haze) для справжнього backdrop blur.

Поза обсягом: інші екрани (Layers, Demolition, Documents, Catalog, Profile, Help) — отримають новий стиль пізніше, через ті самі перевикористовувані компоненти (GlassCard/CommonComponents), без окремого проходу в цій задачі.

## 1. Кольори (`Color.kt`)

Відновлюється бренд-пара з закомічених ресурсів застосунку замість помилково злитих WIP-кольорів:

- `red = #CE1126` — основний акцент: CTA, активні стани, помилки контуру (самоперетин).
- `gold = #C9A227` — окремий, вторинний колір: попередження, виділення, іконка-циркуль на порожніх екранах. **Не дорівнює** `red`.
- `redDeep`, `redMuted`, `redGlow` — похідні від `red` (не від старого `#E30A17`), для тіней/розсіювання.
- OLED темна тема: `graphite #0A0A0C` (глибший чорний — рішення з WIP визнане вдалим і переноситься), `graphiteSoft #16161A`, `graphiteMid #2C2C35`, `graphiteLight #484855`, `textOnDark #FAFAFA`.
- Світла тема: `white #FFFFFF`, `linen #F5F5F7`, `textPrimary #101012`, `textSecondary #6C6C75`, `divider #E5E5EA`.
- Системні: `success #22C55E`, `warning #F59E0B`, `error #EF4444` — без змін.
- **Видаляються** мертві `glassSurfaceLight`/`glassSurfaceDark` — GlassCard більше не будує власний фейковий "скляний" колір, справжнє розмиття дає Haze (див. розділ 2), а базовий колір картки бере звичайний `surface`/`graphiteSoft` з малою альфа (~0.55-0.7) для тонування.

`Theme.kt`: `secondary` для обох тем узгоджується (зараз dark→white, light→textPrimary — різне за призначенням); стає: `secondary = gold` в обох темах (єдине джерело "вторинного" сенсу кольору для Material-інфраструктури).

## 2. GlassCard і справжній blur (Haze)

Додається залежність `dev.chrisbanes.haze` (остання стабільна версія на момент реалізації).

Архітектура:
- Кожен екран, що використовує скляні картки, має один `val hazeState = remember { HazeState() }` на рівні композиції екрана.
- Фоновий шар екрана (те, що має "просвічувати" через скло — фон, інші картки, зображення) позначається `Modifier.hazeSource(hazeState)`.
- `GlassCard` замінює внутрішню реалізацію: замість власного `Brush.verticalGradient` для тіла картки використовує `Modifier.hazeEffect(hazeState, style = HazeStyle(...))` — параметризований кольором тонування (surface color з альфа) і рівнем blur (`blurRadius` за замовчуванням ~16dp).
- Світловідбивна градієнтна рамка (1.2dp) і `shadow` з `spotColor` у відтінку `red` — залишаються без змін, це малюється Compose-модифікаторами над hazeEffect, не залежить від Haze.
- `SurfaceCard` (зараз — тонка обгортка над `GlassCard` лише з іншим `cornerRadius`) видаляється; `cornerRadius` стає звичайним параметром `GlassCard` з дефолтним значенням.
- На екранах без Haze-фону (наприклад, модальні діалоги над системним scrim) `GlassCard` деградує на тонований `surface`-колір без blur-ефекту (немає що розмивати) — це не окремий код-шлях, а природний наслідок відсутності `hazeSource` в ієрархії вище.

## 3. OrnamentalDivider — технічна стилізація

Прибираються ромби, пікселі-квадрати, сателітні крапки і "дужки". Новий вигляд:
- Горизонтальна лінія-hairline (1dp) з градієнтним затуханням від краю до краю: `transparent → red @ 0.5 → transparent`.
- Один акцентний сегмент (4dp × 24dp, колір `red`, скруглені кути) точно по центру лінії — читається як індикатор/маркер, а не орнамент.
- Висота компонента: 8dp (замість поточних 16dp) — менш нав'язливо.
- `ThinDivider` (для таблиць) залишається без змін.

## 4. Типографіка (`Type.kt`)

Без структурних змін до шрифтів (Manrope/Unbounded залишаються). Виправляється конкретний дефект застосування:
- `TabularNumberStyle` і `HeroNumberStyle` мають застосовуватись через `Text(text = ..., style = TabularNumberStyle)` цілим об'єктом `TextStyle`, а не через часткове розбирання окремих полів (`fontFamily = TabularNumberStyle.fontFamily`).
- Аудит усіх місць, де показуються грошові суми/числові колонки (`SummaryScreen`, кошторис, картки кімнат з площею) — переконатись, що всюди застосовано повний стиль з `fontFeatureSettings = "tnum"`.

## 5. Ключові екрани — уніфікація мови картки

Правило для `ProjectsScreen`, `RoomsScreen`, `GeometryScreen`, `SummaryScreen`:
- **Скло (`GlassCard`)**: усі картки контенту — картка проєкту/кімнати в списку, картка підсумку кошторису, картка редактора геометрії.
- **Суцільне (без скла)**: топбар (`CubeMasterTopBar`) — навігаційний якір, не контент; FAB; діалоги (`AlertDialog`/`Dialog`) — модальні поверхні поверх усього, скло тут ускладнює читання; `WarningCard`/`InfoCard` — навмисно суцільні яскраві кольори (жовтий/помаранчевий) для попереджень про небезпеку (areaWarning, openingWarning, гідроізоляція) — скло знижувало б контраст і терміновість сигналу.
- Фон екрана (Scaffold content) отримує `Modifier.hazeSource(hazeState)`, щоб картки могли розмивати те, що під ними (інші картки при скролі, фонові елементи).

## Ризики і обмеження

- Підняття `minSdk` до 31 відсікає Android 10-11 — свідоме рішення користувача, потребує окремого повідомлення в release notes / комунікації з клієнтом (поза обсягом цієї задачі, суто дизайн/код).
- Haze — стороння залежність; версію фіксувати явно в `libs.versions.toml`, перевірити ліцензію (Apache 2.0 — сумісна з комерційним використанням без розкриття джерела, підходить для субпідряду).
- Продуктивність blur на слабких пристроях (типовий користувач — будівельник, не завжди топовий телефон) — перевірити на реальному пристрої (є `verify` skill) після реалізації; якщо помітний jank при скролі списку карток — зменшити `blurRadius` або вимкнути Haze на конкретному важкому екрані (наприклад, довгий список кімнат), залишивши статичну картку.

## Тестування

- Немає automated UI-тестів на цей шар (design-system рівень зазвичай не покривається unit-тестами). Верифікація — вручну через `verify` skill на реальному пристрої: скріншоти світлої/темної теми, перевірка читабельності тексту на склі, перевірка що суми кошторису вирівняні по розряду.
